## 소개

[스킬](https://docs.anthropic.com/en/docs/agents-and-tools/skills)은 특정 작업이나 워크플로우 처리 방법을 가르치는 명령어 세트입니다. 간단한 폴더 형태로 구성됩니다. 스킬은 Claude를 특정 요구사항에 맞게 커스터마이징하는 가장 강력한 방법 중 하나입니다. 매번 대화마다 선호사항, 프로세스, 도메인 전문 지식을 다시 설명하는 대신, 스킬을 사용하면 한 번만 가르치고 매번 효과를 누릴 수 있습니다.

스킬은 반복 가능한 워크플로우가 있을 때 강력합니다. 예를 들어 사양서로부터 프론트엔드 디자인을 생성하거나, 일관된 방법론으로 리서치를 수행하거나, 팀의 스타일 가이드를 따르는 문서를 작성하거나, 다단계 프로세스를 조율하는 작업 등에서 유용합니다. 스킬은 코드 실행 및 문서 생성과 같은 Claude의 기본 기능과 잘 작동합니다. MCP 통합을 구축하는 분들에게는 스킬이 또 다른 강력한 계층을 추가하여 원시 도구 접근을 안정적이고 최적화된 워크플로우로 전환하는 데 도움을 줍니다.

이 가이드는 효과적인 스킬을 구축하는 데 필요한 모든 것을 다룹니다. 계획 및 구조부터 테스트 및 배포까지 포함됩니다. 자신을 위한 스킬을 만들든, 팀을 위한 스킬을 만들든, 커뮤니티를 위한 스킬을 만들든, 전체적으로 실용적인 패턴과 실제 예제를 찾을 수 있습니다.

**배울 내용:**

- 스킬 구조에 대한 기술 요구사항 및 모범 사례
- 독립 실행형 스킬 및 MCP 강화 워크플로우를 위한 패턴
- 다양한 사용 사례에서 잘 작동하는 것으로 확인된 패턴
- 스킬 테스트, 반복 및 배포 방법

**대상 독자:**

- Claude가 특정 워크플로우를 일관되게 따르도록 하고 싶은 개발자
- Claude가 특정 워크플로우를 따르도록 하고 싶은 파워 유저
- 조직 전반에서 Claude가 작동하는 방식을 표준화하려는 팀

**이 가이드를 활용하는 두 가지 경로**

독립 실행형 스킬을 만드는 분이라면 기초, 계획 및 디자인, 카테고리 1-2에 집중하세요. MCP 통합을 강화하는 분이라면 "스킬 + MCP" 섹션과 카테고리 3이 적합합니다. 두 경로 모두 동일한 기술 요구사항을 공유하지만, 사용 사례에 맞는 내용을 선택할 수 있습니다.

**이 가이드에서 얻을 수 있는 것:** 끝까지 읽으면 한 번에 기능적인 스킬을 만들 수 있습니다. skill-creator를 사용하여 첫 번째 작동하는 스킬을 만들고 테스트하는 데 약 15-30분이 소요됩니다.

시작해봅시다.

## 챕터 1: 기초

### 스킬이란 무엇인가?

스킬은 다음을 포함하는 폴더입니다:

- **SKILL.md** (필수): YAML frontmatter가 있는 Markdown 형식의 명령어
- **scripts/** (선택): 실행 가능한 코드 (Python, Bash 등)
- **references/** (선택): 필요에 따라 로드되는 문서
- **assets/** (선택): 출력에 사용되는 템플릿, 폰트, 아이콘

### 핵심 디자인 원칙

#### 점진적 공개 (Progressive Disclosure)

스킬은 3단계 시스템을 사용합니다:

- **1단계 (YAML frontmatter):** 항상 Claude의 시스템 프롬프트에 로드됩니다. 모든 내용을 컨텍스트에 로드하지 않고도 Claude가 각 스킬을 언제 사용해야 하는지 알 수 있을 만큼의 정보만 제공합니다.
- **2단계 (SKILL.md 본문):** Claude가 해당 스킬이 현재 작업과 관련이 있다고 판단하면 로드됩니다. 전체 명령어와 가이드를 포함합니다.
- **3단계 (연결된 파일):** 스킬 디렉토리 내에 번들로 제공되는 추가 파일로, Claude가 필요에 따라 탐색하고 발견할 수 있습니다.

이러한 점진적 공개는 전문성을 유지하면서 토큰 사용량을 최소화합니다.

#### 조합성 (Composability)

Claude는 여러 스킬을 동시에 로드할 수 있습니다. 여러분의 스킬은 다른 스킬과 함께 잘 작동해야 하며, 유일하게 사용 가능한 기능이라고 가정해서는 안 됩니다.

#### 이식성 (Portability)

스킬은 Claude.ai, Claude Code, API 전반에서 동일하게 작동합니다. 한 번 스킬을 만들면 수정 없이 모든 플랫폼에서 작동합니다. 단, 환경이 스킬이 요구하는 종속성을 지원하는 경우에 한합니다.

### MCP 개발자를 위한 안내: 스킬 + 커넥터

> *MCP 없이 독립 실행형 스킬을 만드는 분이라면? 계획 및 디자인으로 건너뛰세요. 나중에 언제든지 여기로 돌아올 수 있습니다.*

이미 [작동하는 MCP 서버](https://modelcontextprotocol.io/)가 있다면, 어려운 부분은 끝났습니다. 스킬은 그 위에 있는 지식 계층입니다. 여러분이 이미 알고 있는 워크플로우와 모범 사례를 포착하여, Claude가 일관되게 적용할 수 있도록 합니다.

#### 주방 비유

**MCP는 전문 주방을 제공합니다:** 도구, 재료, 장비에 접근할 수 있습니다.

**스킬은 레시피를 제공합니다:** 가치 있는 무언가를 만드는 단계별 지침입니다.

함께 작동하면, 사용자가 모든 단계를 스스로 파악하지 않고도 복잡한 작업을 수행할 수 있습니다.

작동 방식:

| MCP (연결성) | 스킬 (지식) |
|---|---|
| Claude를 서비스에 연결합니다 (Notion, Asana, Linear 등) | Claude가 서비스를 효과적으로 사용하는 방법을 가르칩니다 |
| 실시간 데이터 접근 및 도구 호출을 제공합니다 | 워크플로우와 모범 사례를 포착합니다 |
| Claude가 할 수 있는 것 | Claude가 어떻게 해야 하는지 |

#### MCP 사용자에게 중요한 이유

**스킬이 없으면:**

- 사용자가 MCP를 연결하지만 다음에 무엇을 해야 할지 모릅니다
- "통합으로 X를 어떻게 하나요?"라는 지원 티켓이 발생합니다
- 매 대화가 처음부터 시작됩니다
- 사용자가 매번 다르게 프롬프트를 작성하기 때문에 일관되지 않은 결과가 나옵니다
- 실제 문제는 워크플로우 가이드인데 사용자는 커넥터를 탓합니다

**스킬이 있으면:**

- 미리 만들어진 워크플로우가 필요할 때 자동으로 활성화됩니다
- 일관되고 신뢰할 수 있는 도구 사용이 가능합니다
- 모든 상호작용에 모범 사례가 내장됩니다
- 통합에 대한 학습 곡선이 낮아집니다
