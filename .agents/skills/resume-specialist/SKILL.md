# /resume-specialist - 이력서 전문가

프로젝트 이력과 사용자 정보를 기반으로 이력서를 작성, 검토, 최적화합니다.

Codex에서는 `.agents/agents/resume-specialist.md`의 역할 프롬프트를 참고해 직접 수행합니다. 사용자가 명시적으로 하위 에이전트나 병렬 작업을 요청한 경우에만 Codex sub-agent를 사용합니다.

## 데이터 소스

- `src/pages/about.md` - 경력 요약, AI-Native Development 이력
- `src/data/projects.ts` - 프로젝트 목록, 기술 스택, 기간

두 파일이 이중으로 관리되므로 반드시 양쪽을 읽어 최신 정보를 교차 확인하세요.

## 실행 단계

### Step 1: 요청 파악

요청을 다음 중 하나로 분류합니다.

- 작성: 새 이력서/CV 생성
- 검토: 기존 이력서 개선점 분석
- 최적화: ATS 통과율 향상, 키워드 최적화

포지션, 언어, 형식, 대상 국가, JD가 부족하고 결과 품질에 영향을 주면 사용자에게 확인합니다.

### Step 2: 데이터 수집

1. `src/pages/about.md`와 `src/data/projects.ts`를 읽습니다.
2. 사용자가 제공한 JD나 기존 이력서를 함께 검토합니다.
3. 프로젝트 성과는 가능한 한 수치와 영향 중심으로 정리합니다.

### Step 3: 전문 분석/작성

`.agents/agents/resume-specialist.md` 기준으로 다음을 수행합니다.

- 역할: 작성, 검토, 최적화 중 확정된 작업 유형
- 대상 포지션: 사용자가 제공하거나 추론한 포지션
- 프로젝트 이력: Step 2에서 수집한 데이터 요약
- 추가 요구사항: 언어, 형식, 분량, ATS 키워드, 강조할 경험

### Step 4: 결과 제시 및 반복

1. 결과와 핵심 변경 이유를 사용자에게 제시합니다.
2. 수정 방향을 확인합니다.
3. 필요하면 반복 개선합니다.

## 완료 기준

- 이력서 문장은 책임 나열보다 성과와 임팩트를 우선합니다.
- ATS 최적화 시 키워드를 자연스럽게 넣고 과도한 키워드 나열은 피합니다.
- 한국어/영어 이력서 관습을 대상 포지션과 국가에 맞춥니다.
