- 번역투 표현 회피: "~하는 것을 방지할 수 있다" → "~를 막을 수 있다"
- ASCII 아트로 파이프라인 시각화 - 복잡한 워크플로우를 직관적으로 표현
- 유머/위트: 취소선, 괄호 활용으로 자연스러운 유머 표현

### 2026-02-11

날짜: 2026-02-11
포스트: AI에게 번역을 시키면서 깨달은 오케스트레이션의 힘
유형: 승인 (일회성 스타일 변경)
카테고리: ai
파일 경로: contents/blog/ai/ai-translation-orchestration/index.md

피드백 원문: "evan-moon(문동욱) 블로그 스타일로 재작성 요청 → 전문적 지식 전달 스타일로 변경"

특수 상황:
- 이 글은 사용자의 명시적 요청으로 **evan-moon 스타일** (한국 기술 블로거의 특징: 명확한 원리 설명, 스토리텔링 최소화)로 작성됨
- 기존 style-guide.md의 "나" 20회 이상, blockquote 5회 이상 등의 규칙을 **의도적으로 따르지 않음**
- 이것은 **이 글에만 해당하는 일회성 스타일 변경**이므로, style-guide.md의 기존 규칙은 수정하지 않음

추출된 피드백 내용:
1. evan-moon 스타일 재작성 요청
   - 지식 전달 우선 (개인 경험/스토리텔링 최소화)
   - 이론적 배경 → 실제 구현 → 정량적 검증 구조
   - 대상 독자: 기술 원리를 깊이 있게 이해하려는 개발자

2. "조지 밀러의 작업 기억 연구" → "프롬프트 과적(Prompt Overloading)"로 대체
   - 이유: 개발자들이 더 공감할 수 있는 개념
   - 조지 밀러의 매직 넘버 7은 일반적 인지학이지만, 프롬프트 과적은 LLM 사용자의 직접 경험

3. "dependency → 의존성/종속성" 예시 → "context → 맥락/컨텍스트" 예시로 대체
   - 이유: 세 갈래 해석 가능성을 더 명확하게 보여줌
   - dependency는 두 가지만 해석 가능하지만, context는 세 가지 맥락(자연스러운 한국어/음차/API명)을 보여줌

4. Claude Skills 가이드 6부작에 시리즈 페이지 링크 추가
   - `/series/claude-스킬-구축-완벽-가이드/` 링크 추가
   - 품질 모드 섹션에서 실제 사례 언급

5. SoC(Separation of Concerns) 소개 부분은 그대로 유지
   - evan-moon 스타일에서도 알려진 개념을 정식으로 소개하는 패턴 사용
   - 개념 설명 후 구체적 적용으로 진행

최종 점수: 92점 (추정)
