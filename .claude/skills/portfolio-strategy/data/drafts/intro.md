## 즉시 발휘 가능한 역량

윤창식은 React/TypeScript 기반 프론트엔드 영역에서 4년 동안 외주·인하우스를 오가며 AI-Native 워크플로우로 개발 효율을 극대화해왔다. 특히 AI 출력의 비결정성을 디자인 시스템·검증 레이어로 통제하는 패턴이 시그니처다. 팀에 합류하면 다음 역량을 토스 6개 도메인 구분에 맞춰 곧바로 발휘할 수 있다.

| 역량 축 | 토스 도메인 연결고리 | 케이스 스터디 참조 |
|---|---|---|
| AI 파이프라인 설계 (피그마→Tailwind 자동 생성) | Frontend AI Lab / Internal Product | 본문 §1, §5 |
| 모노레포·다중 브랜드 운영 (3개 레포·8+ 앱) | Multi-product 확장 | 본문 §1 |
| 백엔드 의존성 제거 (MSW 선제 스키마) | 가설→제품 빠르게 구현 | 본문 §2 |
| 비표준 빌드 환경 정리 (CRA→Vite 마이그레이션) | Tech Debt / Code Quality Committee | 본문 §3 |
| 오픈소스 도구 제작 (Claude Code 플러그인) | Open Source Committee | 본문 §4 |

두 케이스(§1 kotech-plugin의 스토리북 / §5 슬라이드 자동화의 비주얼 검증)에서 일관되게 드러나는 본인 시그니처는 "AI 출력 비결정성을 디자인 시스템·검증 레이어로 통제"다. 토스 인재상의 안정성·테스트 전략 축에 직접 매칭된다.
