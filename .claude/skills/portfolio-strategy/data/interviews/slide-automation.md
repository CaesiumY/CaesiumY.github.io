# AI-Native · 강의 슬라이드 자동 생성 파이프라인 — 인터뷰 노트

- **인터뷰 일시**: 2026-05-05
- **JD 컨텍스트**: 토스(Viva Republica) 모닥불 FE 전형 — 매칭 축: "Frontend AI Lab + 반복 작업 자동화 + AI 워크플로우 활용"
- **이력서 원문 한 줄**: "강의 슬라이드 자동 생성: 스크립트→PPTX 다단계 파이프라인으로 제작 시간 80% 단축"

## 답변 정리

### 앵커링

- **활용 컨텍스트**: 대학 정규 교과목 14주차 강의 슬라이드 제작에 사용. 이후 다른 업체 특강에도 동일한 방법으로 활용.
- **수동 방식의 P**: 회차당 디자인을 직접 해야 하고, 내용 전달 방식도 깊이 고민해야 해서 시간 소모가 큼.

### 정량 (Result 슬롯)

- **견적**: 약 **2주** (업체에서 제공한 작업 기간)
- **실제 작업 시간**: **14주차 슬라이드 전체를 3일에 완료** (본인 측정)
- 단축 비율: 약 78~85% (이력서의 "80%"는 이 비교에 기반한 표현)

### 의사결정·트레이드오프

- **다단계 파이프라인 설계 철학**: "AI가 아니더라도 인간이 진행하는 단계를 그대로 옮겨왔다." 자연스러운 단계 분해.
- **핵심 의사결정 1 — 디자인 시스템 우선 구축**: 매주차 슬라이드의 일관성 보장을 위해 디자인 시스템을 먼저 구축한 뒤 슬라이드 생성에 진입.
- **핵심 의사결정 2 — PRD 형식 콘텐츠 정리**: 각 슬라이드에 들어갈 정보 + 사용할 화면 구성(예: 그래프, 숫자 비교)을 PRD처럼 문서화한 다음 슬라이드 생성.
- **핵심 의사결정 3 — 비주얼 검증 추가**: 커스텀 폰트 사용 시 AI가 측정한 텍스트 박스 크기가 가변적으로 변경되어 HTML 미리보기 vs PPTX 완성본 사이 괴리감 발생 (텍스트가 박스를 빠져나옴). **스크린샷 + 비주얼 검증 단계** 추가로 해결.

### 운영 측 트레이드오프 (왜 PPTX인가)

- **검토했던 대안**: 슬라이드를 이미지로 직접 생성 (HTML→PNG 변환 후 슬라이드 합성).
- **회피 이유**: 강의 촬영 현장에서 즉석으로 내용 수정해야 하는 경우가 있을 수 있음. 이미지로만 만들면 현장 수정 리스크 발생.
- **채택**: PPTX 형식 유지 → 현장 수정 유연성 확보. 단, AI 출력 품질 통제를 위해 비주얼 검증 추가.

### 본인의 시그니처 패턴 (다른 케이스와 일관)

- "AI 출력의 비결정성을 디자인 시스템·검증 레이어로 통제" — kotech-plugin의 스토리북 검증 패턴과 동일.
- 즉 본인의 의사결정 시그니처: **"AI 출력 → 검증 가능한 추상화 레이어 우선 설계 → 출력 일관성 확보"**.
