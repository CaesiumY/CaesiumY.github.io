# Claude Code 에이전트 가이드

블로그 글쓰기와 번역을 위한 멀티 에이전트 시스템 사용 가이드입니다.

---

## 빠른 선택 가이드

| 작업 | 커맨드 | 스킬 호출 |
|------|--------|----------|
| 블로그 작성 | `/blog-gen "제목"` | → `/blog-writer` |
| 번역 | `/translate-blog <URL>` | → `/translate-writer` |

> **참고**: 커맨드는 해당 스킬을 호출하는 단축키입니다. 동일한 품질 관리 시스템을 거칩니다.

### 워크플로우

```
/blog-gen "제목"  ─────┐
                       ├──▶ /blog-writer 스킬 실행
/blog-writer "제목" ───┘
        │
        ▼
┌─────────────────────────────────────────┐
│  [생성] → [검토] → [자동수정] → [승인]  │
│           (최대 3회)     (사용자 확인)  │
└─────────────────────────────────────────┘
```

## 커맨드 상세

> 커맨드는 스킬의 단축키입니다. 동일한 품질 관리 시스템을 거칩니다.

### /blog-gen

**용도**: 블로그 포스트 작성 (`/blog-writer` 스킬 호출)

**사용법**:
```
/blog-gen "제목"
/blog-gen "React 성능 최적화"
```

**제공 기능** (스킬을 통해):
- ✅ 스타일 가이드 기반 작성
- ✅ 품질 자동 검토 (100점 만점, 80점 이상 통과)
- ✅ 자동 수정 루프 (최대 3회)
- ✅ 사용자 승인 단계
- ✅ 피드백 학습

---

### /translate-blog

**용도**: URL 또는 파일을 한국어로 번역 (`/translate-writer` 스킬 호출)

**사용법**:
```
/translate-blog https://example.com/blog-post
/translate-blog ./local-file.md
```

**제공 기능** (스킬을 통해):
- ✅ 스타일 가이드 + 용어집 참조
- ✅ 번역 품질 자동 검토 (10점 만점, 8점 이상 통과)
- ✅ 자동 수정 루프 (최대 3회)
- ✅ 사용자 승인 단계
- ✅ 피드백 학습 + 용어집 업데이트 제안

## 스킬 상세

### /blog-writer

**용도**: 품질 중심 블로그 작성 (자동 수정 루프 + 학습)

**워크플로우**:
```
┌──────────────────────────────────────────────────────────────┐
│  Phase 0: 스타일 가이드 확인                                   │
│  └── style-guide.md 로드                                      │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  Phase 1: 글 작성 (content-writer)                            │
│  └── 스타일 가이드 준수하여 초안 작성                          │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  Phase 2: 검토 루프 (content-reviewer)                        │
│  └── 100점 만점 평가                                          │
│      ├── 80점 미만: 수정 지시 → Phase 1 (최대 3회)            │
│      └── 80점 이상: 사용자 승인 요청                          │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  Phase 3: 사용자 결정                                         │
│  ├── ✅ 승인 → 저장 + 학습                                    │
│  ├── 📝 수정 요청 → 피드백 반영 후 Phase 1                    │
│  └── ❌ 거절 → 종료                                           │
└──────────────────────────────────────────────────────────────┘
```

**관련 에이전트**:
| 에이전트 | 역할 |
|----------|------|
| style-analyzer | 기존 글 분석 → 스타일 가이드 생성 |
| content-writer | 스타일 가이드 기반 글 작성 |
| content-reviewer | 100점 만점 품질 평가 |
| style-learner | 피드백 학습 → 스타일 가이드 업데이트 |

│  └── ❌ 거절 → 종료                                           │

**관련 에이전트**:
| 에이전트 | 역할 |
|----------|------|

## FAQ

### Q: 커맨드와 스킬의 차이는?

**커맨드** (`/blog-gen`, `/translate-blog`):
- 스킬을 호출하는 **단축키**
- 동일한 품질 관리 시스템 사용

**스킬** (`/blog-writer`, `/translate-writer`):
- 실제 워크플로우가 정의된 곳
- 자동 수정 루프 (점수 통과까지)
- 사용자 승인 단계
- 피드백 학습 → 스타일 가이드 자동 개선

> 💡 커맨드와 스킬 모두 동일한 결과를 제공합니다.

### Q: 스타일 가이드를 수동으로 수정해도 되나요?

네, 가능합니다. 다만:
- 직접 수정 시 `style-history/`에 백업 권장
- 피드백을 통한 학습이 더 일관성 있음

### Q: 새 용어를 용어집에 추가하려면?

1. `/translate-writer` 사용 시 새 용어 발견하면 자동 제안
2. 직접 `.claude/skills/translate-writer/data/glossary.md` 수정 가능

### Q: 검토 점수 기준은?

| 시스템 | 만점 | 통과 기준 |
|--------|------|----------|
| blog-writer | 100점 | 80점 이상 |
| translate-writer | 10점 | 8점 이상 |

### Q: 자동 수정이 3회 반복해도 점수가 안 오르면?

사용자에게 현재 상태를 보고하고, 계속 진행할지 결정을 요청합니다.
