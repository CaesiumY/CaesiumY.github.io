# /polish - 문장 다듬기 커맨드

선택된 문장을 분석하고 개선된 옵션들을 점수와 함께 제시합니다.

## 입력

**대상 텍스트**: $ARGUMENTS

---

## 실행 단계

### Step 1: polish-agent 호출

Task 도구로 **polish-agent** 에이전트를 호출하여 분석 수행:

```
문장: "$ARGUMENTS"
모드: "interactive"
```

에이전트가 반환하는 정보:
- 현재 점수
- 발견된 패턴
- 3-4개 개선 옵션 (점수, 문장, 개선 내용)

### Step 2: 사용자에게 옵션 제시

**AskUserQuestion** 도구를 사용하여 옵션을 제시하세요.

형식:
```
질문: "어떤 옵션으로 수정할까요?"
헤더: "문장 선택"

옵션들:
- 옵션 A (X.X점): "개선된 문장..."
  설명: [수정된 패턴 설명]

- 옵션 B (X.X점): "개선된 문장..."
  설명: [수정된 패턴 설명]

- 옵션 C (X.X점): "개선된 문장..."
  설명: [수정된 패턴 설명]

- 현재 유지 (X.X점): 원본
  설명: 변경 없이 원본 유지
```

### Step 3: 선택 적용

사용자가 옵션을 선택하면:

1. **"현재 유지"** 선택 시:
   - "원본을 유지합니다." 메시지 출력
   - 변경 없이 종료

2. **개선 옵션 선택 시**:
   - 사용자에게 적용 대상 파일 확인
   - Grep 도구로 원본 문장 위치 검색
   - Edit 도구로 선택된 옵션으로 교체
   - 변경 완료 메시지 출력

## 예시 실행

**입력**: "이 기능을 활용하면 성능을 향상시키는 것이 가능합니다"

**polish-agent 분석 결과**:
- 패턴 #1 발견: "~것이 가능합니다" (-0.5)
- 패턴 #10 발견: "활용하면" (-0.2)
- 현재 점수: 9.3점

**옵션 제시 (AskUserQuestion)**:
- 옵션 A (10.0점): "이 기능을 쓰면 성능이 좋아집니다"
  → #1, #10 수정, 간결화
- 옵션 B (9.8점): "이 기능으로 성능을 높일 수 있습니다"
  → #1 수정
- 옵션 C (9.5점): "이 기능을 사용하면 성능 향상이 가능합니다"
  → #10만 수정
- 현재 유지 (9.3점): 원본
